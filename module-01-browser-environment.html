Module 1: Browser Environment & DOM Basics

Understanding what the DOM is and how browsers work

üåê What is the DOM?

DOM = Document Object Model

The DOM is NOT HTML. It's a live, hierarchical object representation of your HTML that the browser creates after parsing. Think of it as a tree structure that JavaScript can interact with to make web pages dynamic and interactive.

Key Point: HTML is static text. The DOM is a living, breathing object tree that can change in real-time!

Exercise 3: How Browsers Render Pages

Understanding the browser rendering pipeline

Parsing HTML

Browser reads HTML sequentially and creates DOM tree

Parsing CSS

Browser reads CSS and creates CSSOM (CSS Object Model)

Render Tree

Combines DOM + CSSOM (only includes visible elements)

Layout (Reflow)

Calculates exact position and size of every element

Paint (Repaint)

Converts layout to actual pixels on screen

Composite

Combines all layers and displays final result

‚ö° Performance Tip: Reflows are expensive! They happen when you change element dimensions, positions, or display. Minimize them by batching DOM updates.

Exercise 7: Window vs Document

Understanding the difference

ü™ü Window Object

  • Represents browser window
  • Global JavaScript object
  • Contains document object
  • Has methods like alert(), confirm()
  • Properties: innerWidth, innerHeight

üìÑ Document Object

  • Represents HTML document
  • Part of window object
  • Contains DOM tree
  • Has DOM manipulation methods
  • Properties: body, head, title
